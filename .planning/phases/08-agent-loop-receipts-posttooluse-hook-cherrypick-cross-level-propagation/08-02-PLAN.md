    1. **Add `compute_rollup_and_emit` to `src-tauri/src/drift/engine.rs`** as a SIBLING of `compute_and_emit` (do NOT modify the existing fn). Signature:
       ```rust
       pub async fn compute_rollup_and_emit(
           app: &tauri::AppHandle,
           uuid: &str,
       ) -> Result<(), String>
       ```
       Implementation:
       - Acquire `DriftLocks::for_uuid(uuid)` — SAME lock as compute_and_emit (RESEARCH.md decision: one lock per UUID, no namespacing — body writes and rollup writes serialize together).
       - Read the node from SQLite: level, rollup_inputs_json, rollup_hash (stored).
       - If level == "L0" → return Ok (L0 exempt; ensure no rollup_derived row exists for L0 — DELETE if found).
       - If `rollup_inputs_json` is NULL or `[]` → state = `untracked`. Upsert `rollup_derived` row with state='untracked', stored_rollup_hash=NULL, computed_rollup_hash=''. Emit `rollup:changed` only if state changed since last computation. Return.
       - Otherwise, parse `rollup_inputs` JSON into `Vec<RollupInput>`. For each `RollupInput { child_uuid, sections }`:
         - Read child node's `section_hashes_json` from `nodes` table.
         - If column is NULL (v2 sidecar that hasn't been re-written), READ the child sidecar body from disk via the `read_sidecar_file(repo, child_uuid)` helper added by 08-01 (sidecar/frontmatter.rs), call `crate::sidecar::section_parser::compute_section_hashes(&body)` — Pitfall 7 lazy migration: in-memory only.
         - For each cited section name in `sections`, look up its sha256 in the child's section_hashes. Append `(child_uuid, section_name, sha256)` to a sorted list.
       - Concatenate the sorted list as `format!("{child}:{section}={hash};")`-style string and compute `sha256` over it → this is `computed_rollup_hash`.
       - Compare to `stored_rollup_hash` (from `rollup_derived.stored_rollup_hash`, OR fallback to `nodes.rollup_hash` if rollup_derived has no row yet — first-pass case).
       - state = if computed == stored → `fresh`, else → `stale`.
       - Upsert `rollup_derived` row with `(uuid, computed_rollup_hash, stored_rollup_hash, state, generation_at_check, checked_at)`.
       - On state change (compare old state from DB with new), emit `app.emit("rollup:changed", json!({"uuid": uuid, "state": state, "generation": generation_at_check}))`.

       NOTE on `stored_rollup_hash` source: PROPAGATION.md decision in RESEARCH.md Pattern 2 — sidecar holds the committed `rollup_hash` field; rollup_derived holds detection state. On first-pass detection where rollup_derived has no row, read `nodes.rollup_hash` (set during a prior real reconcile commit) as the stored value. NULL means "never reconciled" → state is `stale` if rollup_inputs has any entries with non-empty section_hashes mismatch (or if rollup_inputs is non-empty and rollup_hash has never been committed → start as `stale` so user sees the gap).
