<objective>
Land the contract-anchored retrieval algorithm — the demo-load-bearing claim that the system "isn't just RAG over chat" (Beat 2 narration anchor).

Three-stage pipeline:
1. **Lineage scope** — recursive CTE walking parent_uuid edges from the contract atom UP TO L0 + sibling atoms (children of the same parent). Cousins (children of ancestors other than parent) EXCLUDED per CONTEXT lock.
2. **Candidate selection** — FTS5 over substrate_nodes_fts.applies_when (BM25 ranked) + optional embedding cosine; RRF (k=60) merges; top-15 returned. Zero-hit fallback to global semantic match if scope returns <5 candidates.
3. **LLM rerank** — listwise prompt over top-15 → top-5 via single `claude -p --bare` call; defensive parser tolerates code-fence wrapping + invalid indices.

Purpose:
- Plan 11-04 (Delegate button) calls `find_substrate_for_atom(scope_uuid, contract_body, 5, with_rerank=true)` to populate the composing overlay's 5 hits.
- Plan 11-05's UAT gates SC 4 (top-3 retrieval on 50-constraint substrate) — this plan implements the algorithm; Plan 11-05's UAT measures it end-to-end against the seeded substrate.
- Phase 9 mass-edit ranking will reuse `candidate_selection()` (Open Question 4 share-the-candidate-step recommendation).

**Critical: cousins EXCLUDED.** Lineage scope = parent + ancestors-up-to-L0 + siblings only. CONTEXT.md narrows this explicitly: "the chain of contracts that govern this work" — cousin atoms (children of ancestors at the same level as the parent) are not part of that chain. Phase 11 ships strict scope; revisit only if precision feels too narrow in practice (Deferred Idea: cousin-scoped retrieval).

**Critical: --bare on every claude -p invocation.** Without --bare, MCP discovery + CLAUDE.md + skills add 1-3s of startup latency and non-deterministic context. Distiller (Plan 11-02) and rerank (this plan) both need fast deterministic context. Anti-pattern explicitly called out in 11-RESEARCH.md.

**Critical: rerank only on Delegate dispatch.** This plan's rerank function is called ONCE per Delegate dispatch (Plan 11-04). MCP tools (Plan 11-02 find_constraints_for_goal etc.) DO NOT call rerank — they're hot paths called by agents during reasoning loops; cost discipline matters. The split is enforced by the architecture: rerank lives in retrieval/rerank.rs and is NOT exposed via MCP.

Output: retrieval module with three submodules + commands/retrieval.rs (private find_substrate_for_atom) + retrieval_unit_tests.rs covering lineage walker, RRF, defensive index parser. No Tauri commands exposed via MCP from this plan (Delegate UI in Plan 11-04 calls the private Rust IPC).

    use sqlx::SqlitePool;

    /// Walk parent_uuid edges from `scope_uuid` UP TO L0 + collect siblings (same parent).
    /// EXCLUDES cousins per CONTEXT lock.
    ///
    /// Returns: Vec of uuids covering parent + all ancestors + siblings (NOT including scope_uuid itself).
    pub async fn lineage_scope_uuids(
        pool: &SqlitePool,
        scope_uuid: &str,
    ) -> Result<Vec<String>, String> {
        // Two CTEs:
        // 1. ancestors — recursive walk via parent_uuid
        // 2. siblings — children of the target's parent_uuid (excluding target itself)
        // UNION the two and exclude scope_uuid from the result.
        let rows: Vec<(String,)> = sqlx::query_as(
            r#"
            WITH RECURSIVE ancestors(uuid, parent_uuid) AS (
                SELECT uuid, parent_uuid FROM nodes WHERE uuid = ?1
                UNION ALL
                SELECT n.uuid, n.parent_uuid
                FROM nodes n
                JOIN ancestors a ON n.uuid = a.parent_uuid
                WHERE a.parent_uuid IS NOT NULL
            )
            SELECT uuid FROM ancestors WHERE uuid != ?1
            UNION
            SELECT s.uuid
            FROM nodes s
            JOIN nodes target ON target.uuid = ?1
            WHERE s.parent_uuid IS NOT NULL
              AND s.parent_uuid = target.parent_uuid
              AND s.uuid != ?1
            "#,
        )
        .bind(scope_uuid)
        .fetch_all(pool)
        .await
        .map_err(|e| format!("lineage walk: {e}"))?;

        Ok(rows.into_iter().map(|r| r.0).collect())
    }
    ```

    **Note on cousin exclusion:** the SQL above does NOT walk siblings of ancestors. The `siblings` clause only returns children of `target.parent_uuid` — i.e., siblings at the SAME level as `scope_uuid`. Children of ancestors other than parent (cousins) never enter the result set. This matches the CONTEXT lock.

    ```rust
    use crate::distiller::types::SubstrateNode;
    use crate::retrieval::{SubstrateHit, ScopeUsed};
    use sqlx::SqlitePool;
    use std::collections::HashMap;

    /// Top-N candidate selection via FTS5 + optional embedding cosine + RRF (k=60).
    /// Returns up to `limit` hydrated SubstrateHit rows.
    ///
    /// `lineage_scope_uuids` is currently used as TEXT CONTEXT in the rerank prompt rather than a
    /// hard graph filter (Pitfall 2: substrate is keyed by source_session_id, not contract uuids;
    /// no JOIN is possible in v1). This function therefore IGNORES scope_uuids for hard filtering
    /// and uses them only for zero-hit-fallback decision: if FTS5 over the query returns <5 hits,
    /// fall back to global match across ALL current-truth substrate nodes.
    ///
    /// PUBLIC API: Phase 9 mass-edit will reuse this function (Open Question 4).
    pub async fn candidate_selection(
        pool: &SqlitePool,
        scope_uuids: &[String],
        query: &str,
        query_embedding: Option<&[f32]>,
        limit: usize,
    ) -> Result<Vec<SubstrateHit>, String> {
        // FTS5 candidates (cheap, broad)
        let fts_rows: Vec<SubstrateNode> = sqlx::query_as::<_, SubstrateNode>(
            r#"
            SELECT s.uuid, s.node_type, s.text, s.scope, s.applies_when,
                   s.source_session_id, s.source_turn_ref, s.source_quote, s.source_actor,
                   s.valid_at, s.invalid_at, s.expired_at, s.created_at,
                   s.confidence, s.episode_id, s.invalidated_by
            FROM substrate_nodes_fts fts
            JOIN substrate_nodes s ON s.uuid = fts.uuid
            WHERE substrate_nodes_fts MATCH ?
              AND s.invalid_at IS NULL
            ORDER BY fts.rank
            LIMIT ?
            "#,
        )
        .bind(query)
        .bind((limit * 2) as i64)
        .fetch_all(pool)
        .await
        .map_err(|e| format!("fts candidates: {e}"))?;

        // RRF combination — single source for now (FTS5); embedding source layered if query_embedding provided
        let mut combined: HashMap<String, (f64, SubstrateNode)> = HashMap::new();
        for (rank, node) in fts_rows.into_iter().enumerate() {
            let rrf = 1.0 / (60.0 + rank as f64 + 1.0);
            combined.entry(node.uuid.clone()).or_insert((0.0, node)).0 += rrf;
        }

        if let Some(qe) = query_embedding {
            let cosine_rows = compute_cosine_top_n(pool, qe, limit * 2).await?;
            for (rank, (uuid, node)) in cosine_rows.into_iter().enumerate() {
                let rrf = 1.0 / (60.0 + rank as f64 + 1.0);
                combined.entry(uuid).or_insert((0.0, node)).0 += rrf;
            }
        }

        // Sort by combined RRF score descending
        let mut candidates: Vec<(String, f64, SubstrateNode)> = combined
            .into_iter()
            .map(|(uuid, (score, node))| (uuid, score, node))
            .collect();
        candidates.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap_or(std::cmp::Ordering::Equal));
        candidates.truncate(limit);

        // Zero-hit fallback (CONTEXT lock): if scope+FTS5 returns <5, fall back to global semantic match.
        // For v1 we treat "scope" as informational only (Pitfall 2), so the fallback is "if FTS5 returned
        // <5, mark scope_used as Broad". A more sophisticated v2 may re-issue with a broadened query.
        let scope_used = if candidates.len() < 5 && !scope_uuids.is_empty() {
            ScopeUsed::Broad
        } else {
            ScopeUsed::Lineage
        };

        Ok(candidates
            .into_iter()
            .map(|(_uuid, _score, node)| SubstrateHit::from_node(node, scope_used.clone()))
            .collect())
    }

    /// Cosine over substrate_embeddings BLOBs. Loads ALL current-truth embeddings into memory
    /// and computes cosine in Rust. At 50-constraint scale, sub-1ms.
    async fn compute_cosine_top_n(
        pool: &SqlitePool,
        query: &[f32],
        limit: usize,
    ) -> Result<Vec<(String, SubstrateNode)>, String> {
        // Load all (uuid, vector, full node row) tuples for current-truth nodes
        let rows: Vec<(String, Vec<u8>)> = sqlx::query_as(
            r#"
            SELECT e.uuid, e.vector
            FROM substrate_embeddings e
            JOIN substrate_nodes n ON n.uuid = e.uuid
            WHERE n.invalid_at IS NULL
            "#,
        )
        .fetch_all(pool)
        .await
        .map_err(|e| format!("embeddings load: {e}"))?;

        let mut scored: Vec<(String, f32)> = rows
            .into_iter()
            .filter_map(|(uuid, blob)| {
                let vec = blob_to_f32(&blob)?;
                if vec.len() != query.len() {
                    return None; // dimension mismatch (Pitfall 4) — silently skip
                }
                let cos = cosine(query, &vec);
                Some((uuid, cos))
            })
            .collect();

        scored.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap_or(std::cmp::Ordering::Equal));
        scored.truncate(limit);

        // Hydrate full SubstrateNode rows
        let uuids: Vec<String> = scored.iter().map(|(u, _)| u.clone()).collect();
        if uuids.is_empty() {
            return Ok(vec![]);
        }

        // Build IN placeholders
        let placeholders = std::iter::repeat("?").take(uuids.len()).collect::<Vec<_>>().join(",");
        let q = format!(
            "SELECT s.uuid, s.node_type, s.text, s.scope, s.applies_when,
                    s.source_session_id, s.source_turn_ref, s.source_quote, s.source_actor,
                    s.valid_at, s.invalid_at, s.expired_at, s.created_at,
                    s.confidence, s.episode_id, s.invalidated_by
             FROM substrate_nodes s
             WHERE s.uuid IN ({placeholders})"
        );
        let mut q = sqlx::query_as::<_, SubstrateNode>(&q);
        for u in &uuids {
            q = q.bind(u);
        }
        let nodes: Vec<SubstrateNode> = q.fetch_all(pool).await.map_err(|e| format!("hydrate: {e}"))?;
        let by_uuid: HashMap<String, SubstrateNode> = nodes.into_iter().map(|n| (n.uuid.clone(), n)).collect();

        // Preserve cosine ordering
        Ok(scored
            .into_iter()
            .filter_map(|(uuid, _score)| by_uuid.get(&uuid).cloned().map(|n| (uuid, n)))
            .collect())
    }

    fn blob_to_f32(blob: &[u8]) -> Option<Vec<f32>> {
        if blob.len() % 4 != 0 {
            return None;
        }
        Some(blob.chunks_exact(4)
            .map(|chunk| {
                let bytes: [u8; 4] = chunk.try_into().unwrap();
                f32::from_le_bytes(bytes)
            })
            .collect())
    }

    fn cosine(a: &[f32], b: &[f32]) -> f32 {
        let mut dot = 0.0_f32;
        let mut na = 0.0_f32;
        let mut nb = 0.0_f32;
        for i in 0..a.len() {
            dot += a[i] * b[i];
            na += a[i] * a[i];
            nb += b[i] * b[i];
        }
        if na == 0.0 || nb == 0.0 {
            return 0.0;
        }
        dot / (na.sqrt() * nb.sqrt())
    }
    ```

    /// out-of-bounds indices + duplicates.
    ///
    /// Falls back to original FTS5 ordering if rerank parse fails entirely (insurance against LLM jitter).
    pub async fn llm_rerank(
        app: &AppHandle,
        contract_body: &str,
        candidates: &[SubstrateHit],
        top_k: usize,
    ) -> Result<Vec<SubstrateHit>, String> {
        if candidates.is_empty() {
            return Ok(vec![]);
        }
        if candidates.len() <= top_k {
            return Ok(candidates.to_vec());
        }

        // Pitfall 8: truncate contract_body to 800 chars
        let body_truncated = if contract_body.chars().count() > 800 {
            contract_body.chars().take(800).collect::<String>()
        } else {
            contract_body.to_string()
        };

        let candidates_text = candidates
            .iter()
            .enumerate()
            .map(|(i, c)| {
                let aw = c.applies_when.as_deref().unwrap_or("");
                format!("[{i}] type={} text=\"{}\" applies_when=\"{}\"", c.node_type, c.text, aw)
            })
            .collect::<Vec<_>>()
            .join("\n");

        let prompt = format!(
            r#"You are reranking substrate rules for a coding agent's task.

Contract body (the work to do):
<contract>
{body_truncated}
</contract>

Candidates (top-{n} from semantic search):
<candidates>
{candidates_text}
</candidates>

Task: Pick the {top_k} candidates whose `applies_when` MOST DIRECTLY constrains how the agent should
implement this contract. Order them most-relevant first.

Output ONLY a JSON array of indices like [3, 7, 1, 5, 2]. No commentary."#,
            n = candidates.len(),
        );

        let output = app
            .shell()
            .command("claude")
            .args(["-p", &prompt, "--output-format", "json", "--bare"])
            .output()
            .await
            .map_err(|e| format!("rerank claude spawn: {e}"))?;

        if !output.status.success() {
            // Fallback: original ordering
            tracing::warn!("rerank claude exit non-zero; falling back to FTS5 order");
            return Ok(candidates.iter().take(top_k).cloned().collect());
        }

        let response: serde_json::Value = serde_json::from_slice(&output.stdout)
            .map_err(|e| format!("response parse: {e}"))?;
        let result_text = response
            .get("result")
            .and_then(|v| v.as_str())
            .unwrap_or("[]")
            .to_string();

        let indices = parse_indices_defensive(&result_text);

        // Filter: drop out-of-bounds + duplicates
        let mut seen = std::collections::HashSet::new();
        let mut ordered: Vec<SubstrateHit> = indices
            .into_iter()
            .filter(|&i| i < candidates.len() && seen.insert(i))
            .map(|i| candidates[i].clone())
            .take(top_k)
            .collect();

        // Backfill from original FTS5 ordering for any missing slots (insurance)
        if ordered.len() < top_k {
            for (i, c) in candidates.iter().enumerate() {
                if seen.contains(&i) {
                    continue;
                }
                ordered.push(c.clone());
                seen.insert(i);
                if ordered.len() >= top_k {
                    break;
                }
            }
        }

        Ok(ordered)
    }

    /// Defensive index parser: handles raw JSON array, code-fence-wrapped JSON, and adjacent
    /// preamble. Returns Vec<usize>; empty Vec if all parses fail.
    pub fn parse_indices_defensive(text: &str) -> Vec<usize> {
        // Attempt 1: direct parse
        if let Ok(indices) = serde_json::from_str::<Vec<i64>>(text) {
            return indices.into_iter().filter(|i| *i >= 0).map(|i| i as usize).collect();
        }
        // Attempt 2: strip code fences + whitespace
        let stripped = text.trim().trim_start_matches("```json").trim_start_matches("```").trim_end_matches("```").trim();
        if let Ok(indices) = serde_json::from_str::<Vec<i64>>(stripped) {
            return indices.into_iter().filter(|i| *i >= 0).map(|i| i as usize).collect();
        }
        // Attempt 3: regex-extract first JSON-array-like substring
        if let Some(start) = stripped.find('[') {
            if let Some(end) = stripped[start..].find(']') {
                let candidate = &stripped[start..=start + end];
                if let Ok(indices) = serde_json::from_str::<Vec<i64>>(candidate) {
                    return indices.into_iter().filter(|i| *i >= 0).map(|i| i as usize).collect();
                }
            }
        }
        Vec::new()
    }
    ```
