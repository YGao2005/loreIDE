1. Acquire pool via the project's standard helper (see how `commands/substrate.rs` does it — likely `pool_from_app(&app)` or equivalent).
2. Read the old row in a single SELECT WHERE uuid=?1 AND invalid_at IS NULL — fetch_optional → if None, return Err("rule {uuid} not found or already tombstoned"). Capture the old `text` and `applies_when` for the audit row's `before_text` field. (Concatenate text + " | applies_when: " + applies_when into before_text per plan 15-01 audit spec — store the concatenation so a single column captures the editable surface; alternatively store text only and accept that applies_when changes are not in the audit before/after columns. PICK text-only for v1; document the choice.)
3. Generate `new_uuid = uuid::Uuid::new_v4().to_string()`.
4. Capture `now = chrono::Utc::now().to_rfc3339()`.
5. `let mut tx = pool.begin().await?;`
6. INSERT new row via `INSERT INTO substrate_nodes (uuid, node_type, text, scope, applies_when, source_session_id, source_turn_ref, source_quote, source_actor, valid_at, invalid_at, expired_at, created_at, confidence, episode_id, invalidated_by, anchored_uuids, prev_version_uuid) SELECT ?new_uuid, node_type, ?new_text, scope, ?new_applies_when, source_session_id, source_turn_ref, source_quote, source_actor, ?now, NULL, NULL, ?now, confidence, episode_id, NULL, anchored_uuids, ?old_uuid FROM substrate_nodes WHERE uuid = ?old_uuid` — copies all columns from old row, overrides text/applies_when/timestamps/prev_version_uuid.
7. UPDATE old row: `UPDATE substrate_nodes SET invalid_at = ?now, invalidated_reason = ?invalidated_reason WHERE uuid = ?old_uuid AND invalid_at IS NULL` where `invalidated_reason = format!("refined: {}", reason)`.
8. INSERT into substrate_edits: edit_id=new_v4, rule_uuid=new_uuid, prev_version_uuid=old_uuid, new_version_uuid=new_uuid, actor=actor, edited_at=now, before_text=old_text, after_text=new_text, reason=reason, kind='refine'.
9. `tx.commit().await?;`
10. Return Ok(new_uuid).

The FTS triggers from plan 15-01 fire automatically inside the transaction — the new row is INSERTed (trigger adds it to FTS), the old row is UPDATEd (trigger removes it from FTS and skips re-INSERT because invalid_at IS NOT NULL).

Implementation: Use the recursive CTE from RESEARCH §Pattern 3 Option A (lines ~120-128 of 15-RESEARCH.md):

```sql
WITH RECURSIVE chain(uuid, text, applies_when, valid_at, invalid_at, invalidated_reason, prev_version_uuid, depth) AS (
    SELECT uuid, text, applies_when, valid_at, invalid_at, invalidated_reason, prev_version_uuid, 0
    FROM substrate_nodes WHERE uuid = ?1
    UNION ALL
    SELECT s.uuid, s.text, s.applies_when, s.valid_at, s.invalid_at, s.invalidated_reason, s.prev_version_uuid, c.depth+1
    FROM substrate_nodes s JOIN chain c ON s.uuid = c.prev_version_uuid
    WHERE c.depth < 50
)
SELECT chain.*, se.actor, se.before_text, se.reason
FROM chain
LEFT JOIN substrate_edits se ON se.new_version_uuid = chain.uuid AND se.kind = 'refine'
ORDER BY chain.valid_at ASC
```

The LEFT JOIN on substrate_edits surfaces the actor + reason that produced each version. version_number is 1-indexed: assign in Rust after the query result is sorted by valid_at ASC.
