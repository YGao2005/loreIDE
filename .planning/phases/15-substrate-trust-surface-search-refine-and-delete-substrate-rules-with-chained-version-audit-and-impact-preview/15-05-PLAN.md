**list_tombstoned_rules** (returns the chain-head tombstones per RESEARCH Pitfall 5):

```rust
#[derive(serde::Serialize)]
pub struct TombstonedRule {
    pub uuid: String,
    pub name: String,                    // first line of text
    pub kind: String,                    // node_type
    pub text: String,
    pub invalidated_reason: Option<String>,
    pub invalidated_at: Option<String>,  // == invalid_at
    pub invalidated_by: Option<String>,  // actor
}

#[tauri::command]
pub async fn list_tombstoned_rules(app: tauri::AppHandle) -> Result<Vec<TombstonedRule>, String>
```

SQL:
```sql
SELECT uuid, node_type AS kind, text, invalidated_reason, invalid_at AS invalidated_at, invalidated_by
FROM substrate_nodes
WHERE invalid_at IS NOT NULL
  AND uuid NOT IN (
      SELECT prev_version_uuid FROM substrate_nodes
      WHERE prev_version_uuid IS NOT NULL AND invalid_at IS NULL
  )
ORDER BY invalid_at DESC
LIMIT 100
```

WHY this query: it returns rows where (a) invalid_at is set AND (b) no LATER row points back to this uuid via prev_version_uuid AND has an active head. This filters out mid-chain tombstones (a→b→c where a and b are tombstoned and c is active — only c's predecessor chain is hidden; if all are tombstoned, only the latest is shown). Per RESEARCH Pitfall 5: "Substrate Health view only shows the CURRENT head of each chain (most recent invalid_at IS NOT NULL row, where prev_version_uuid points at no later active row)."

Compute `name` in Rust by calling the existing `first_line(&text, &uuid)` helper from `commands/substrate.rs` (re-export it via `pub` if needed; avoid duplicating).

**restore_substrate_rule**:

```rust
#[tauri::command]
pub async fn restore_substrate_rule(
    app: tauri::AppHandle,
    uuid: String,
    actor: String,  // "human:<email>"
) -> Result<(), String>
```

Implementation:
1. Read the row: SELECT uuid, text, invalid_at FROM substrate_nodes WHERE uuid=?1. If None, Err("rule not found"). If invalid_at IS NULL, Err("rule is already active — nothing to restore").
2. Active-successor guard: SELECT COUNT(*) FROM substrate_nodes WHERE prev_version_uuid=?1 AND invalid_at IS NULL. If >0, Err("cannot restore: chain has an active successor — restore would create two heads").
3. now = chrono::Utc::now().to_rfc3339(); capture current text for after_text in audit.
4. tx.begin().
5. UPDATE substrate_nodes SET invalid_at=NULL, invalidated_reason=NULL, invalidated_by=NULL WHERE uuid=?1.
6. INSERT INTO substrate_edits (edit_id, rule_uuid, prev_version_uuid, new_version_uuid, actor, edited_at, before_text, after_text, reason, kind) VALUES (new_v4, uuid, NULL, NULL, actor, now, NULL, current_text, format!("restored by {actor}"), 'restore').
7. tx.commit(). The FTS UPDATE trigger fires — removes the old (tombstoned) entry from FTS and re-INSERTs because invalid_at IS NULL post-update.
8. Return Ok(()).
