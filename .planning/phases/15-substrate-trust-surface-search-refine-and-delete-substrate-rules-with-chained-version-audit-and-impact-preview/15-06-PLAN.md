```tsx
import { useCitationStore } from '@/store/citation';
import { useVerifierStore } from '@/store/verifier';

// inside the click handler:
const onFlagClick = (row: VerifierRow) => {
  // 1. Register the commit-handshake. SourceArchaeologyModal (15-03) invokes this
  //    callback AFTER a successful refine, with the ORIGINAL uuid (the rule that was
  //    refined, before re-pointing to the new chain head). closeCitation will clear
  //    this callback automatically (15-03 Step 0 wired that), so a subsequent Cmd+P
  //    open without setting onRefineSuccess leaves the field null and a refine from
  //    THAT path has no panel side-effects.
  useCitationStore.getState().setOnRefineSuccess((uuid) => {
    useVerifierStore.getState().acceptFlag(uuid);
  });

  // 2. Open the modal on the rule. Preserve plan 13-09's 8s halo on parent screen
  //    card — only ADD the two store mutations; do not remove existing behaviour.
  useCitationStore.getState().openCitationUuid(row.ruleUuid);
};
```

WHY this approach: the previous draft of this plan inferred "refine happened" by polling `getSubstrateChain` after every modal close and checking chain length > 1. That was post-hoc inference + a DB round-trip on every modal close + fragile under unexpected close paths (background click, Escape, etc.). The callback approach is direct: SourceArchaeologyModal fires the handshake EXACTLY when a refine commits, regardless of how the modal subsequently closes. No useEffect, no chain-length probe, no DB roundtrip on close.

WHY the wire is generic (`row.ruleUuid` not a hardcoded UUID): the seeded UUID for Beat 3 is `con-settings-no-modal-interrupts-2025-Q4`, but the click handler must be generic so any future flag routes correctly. The seed-specific binding lives in the seed file, not in the component.

WHY the silence guarantee for Cmd+P is automatic: 15-03's `closeCitation` clears `onRefineSuccess` to null on every modal close. Any subsequent open (including Cmd+P substrate hits routed via `useCitationStore.openCitationUuid` in plan 15-02 Task 2) inherits the null callback unless the opener explicitly sets it. VerifierPanel is the only opener that registers a callback in v1.
