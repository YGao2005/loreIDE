<objective>
Build the Rust-side derivation pipeline for Phase 6: a non-blocking `derive_contracts` Tauri command that, for each requested uuid, computes a fresh `code_hash` over the node's source ranges, skips when unchanged (DERIVE-02) or when the user has pinned the contract (DERIVE-03), calls the Anthropic Messages API to generate a natural-language contract body (DERIVE-01), then atomically rewrites the sidecar `.md` and upserts SQLite via the existing Phase 2 write path. Progress is emitted per-uuid via `derive:progress` Tauri events so the Phase 3 / Phase 4 UI stays non-blocking.

Purpose: This plan is the first code in the repo that actually computes `code_hash` — every node in SQLite today has `code_hash = NULL` because the scanner passes `fm.code_hash` from sidecar frontmatter (which is NULL for all hand-authored fixtures). Phase 7's drift detection is a hard consumer of `code_hash` baselines, so getting the hash semantics right here is load-bearing for two downstream phases. Keeping the entire derivation pipeline inside Rust (rather than shelling to the MCP sidecar or the TypeScript path) gives us: (a) direct access to `RepoState`, `DbInstances`, and the existing `write_sidecar` / `upsert_node_pub` helpers; (b) reuse of `sha2` and `hex` already in Cargo.toml; (c) `tauri::async_runtime::spawn` + `app.emit` as the established progress pattern from Phase 5's `mcp:status`.

Output: Frontend can `await invoke('derive_contracts', { uuids: ['abc-...'] })` and immediately move on; within seconds the matching `derive:progress` event(s) arrive and (on `status: done`) the sidecar file has an LLM-generated body plus baseline hashes. Plan 06-02 layers the UI (button + progress display + refresh) on top of this surface.

//! Phase 6 contract derivation pipeline (DERIVE-01 / DERIVE-02 / DERIVE-03).
//!
//! This file owns the pure helpers and the LLM HTTP call. The Tauri command
//! handler + event emission + sidecar write + SQLite upsert orchestration lives
//! below in `derive_contracts` (Task 2 of Plan 06-01).
//!
//! Hash semantics reminder (Pitfall 2 in 06-RESEARCH.md):
//!   - `code_hash`    = SHA-256 over the concatenated lines described by
//!                       `code_ranges` at DERIVATION TIME. Phase 7's drift
//!                       detection recomputes this and compares to the stored
//!                       value — drift = source changed since derivation.
//!   - `contract_hash` = SHA-256 over the contract BODY text at derivation
//!                       time. Used later to detect user edits after
//!                       derivation.
//!
//! Both are set at the SAME moment post-derivation, producing a baseline that
//! Phase 7 compares against future recomputed values. They will typically be
//! different values (one hashes source, one hashes prose) — that is expected.

use std::path::Path;

use sha2::{Digest, Sha256};

use crate::sidecar::frontmatter::CodeRange;

/// Compute SHA-256 hex over the SOURCE LINES a node covers (not the whole file).
///
/// Two nodes pointing at different functions in the same file get independent
/// hashes — this is the drift-detection granularity we need.
///
/// Returns `None` if ANY of the referenced files are unreadable (caller should
/// treat as "cannot compute — skip derivation gracefully"). `end_line` is
/// clamped to the actual file length so a sidecar written before a refactor
/// that shrank the file does not panic.
pub fn compute_code_hash(repo_path: &Path, code_ranges: &[CodeRange]) -> Option<String> {
    if code_ranges.is_empty() {
        return None;
    }
    let mut hasher = Sha256::new();
    for range in code_ranges {
        let file_path = repo_path.join(&range.file);
        let content = std::fs::read_to_string(&file_path).ok()?;
        let lines: Vec<&str> = content.lines().collect();
        let start = (range.start_line as usize).saturating_sub(1);
        let end = (range.end_line as usize).min(lines.len());
        if start >= end {
            continue;
        }
        for line in &lines[start..end] {
            hasher.update(line.as_bytes());
            hasher.update(b"\n");
        }
    }
    Some(hex::encode(hasher.finalize()))
}

/// Compute SHA-256 hex over the contract body text (trimmed).
///
/// Body trimming matches the comparison that Phase 7 drift detection will
/// perform — a leading / trailing newline inserted by an editor must not
/// trigger a spurious drift signal.
pub fn compute_contract_hash(body: &str) -> String {
    let mut hasher = Sha256::new();
    hasher.update(body.trim().as_bytes());
    hex::encode(hasher.finalize())
}

/// Extract the source snippet a node covers, capped at `max_lines` lines total
/// across all ranges. Keeps the LLM prompt within a reasonable token budget.
/// When truncation happens, appends a `// ... N lines truncated` marker so the
/// LLM knows the snippet is partial.
pub fn extract_source(
    repo_path: &Path,
    code_ranges: &[CodeRange],
    max_lines: usize,
) -> String {
    let mut out = String::new();
    let mut collected = 0usize;
    let mut truncated = 0usize;

    for range in code_ranges {
        if collected >= max_lines {
            truncated += (range.end_line as usize).saturating_sub(
                (range.start_line as usize).saturating_sub(1),
            );
            continue;
        }
        let file_path = repo_path.join(&range.file);
        let Ok(content) = std::fs::read_to_string(&file_path) else {
            continue;
        };
        let lines: Vec<&str> = content.lines().collect();
        let start = (range.start_line as usize).saturating_sub(1);
        let end = (range.end_line as usize).min(lines.len());
        if start >= end {
            continue;
        }

        out.push_str(&format!("// {}\n", range.file));
        for line in &lines[start..end] {
            if collected >= max_lines {
                truncated += end - (start + collected);
                break;
            }
            out.push_str(line);
            out.push('\n');
            collected += 1;
        }
        out.push('\n');
    }

    if truncated > 0 {
        out.push_str(&format!("// ... {truncated} lines truncated\n"));
    }
    out
}

/// Anthropic Messages API client call.
///
/// Source: https://docs.anthropic.com/en/api/messages (verified in
/// 06-RESEARCH.md 2026-04-24). Returns the assistant text, trimmed.
///
/// NOTE: Callers must reuse the `Client` across batch calls (Pitfall in
/// 06-RESEARCH.md Anti-Patterns). Do NOT construct a new Client inside this
/// function — accept one from the caller.
pub async fn call_anthropic_for_contract(
    client: &reqwest::Client,
    api_key: &str,
    source_snippet: &str,
    uuid: &str,
    file_label: &str,
    level_label: &str,
    kind_label: &str,
) -> anyhow::Result<String> {
    let system_prompt = "You are a contract writer for a Contract IDE. \
        Given a source code snippet, write a concise 2-4 sentence natural-language \
        contract body describing: (1) what this code does (behaviour, not \
        implementation), (2) key inputs and outputs, (3) any invariants it \
        maintains, (4) its role in the larger system. Output ONLY the contract \
        body as plain prose — no YAML, no code fences, no headers.";

    let user_prompt = format!(
        "File: {file_label}\nNode UUID: {uuid}\nLevel: {level_label} ({kind_label})\n\n```\n{source_snippet}\n```"
    );

    let resp = client
        .post("https://api.anthropic.com/v1/messages")
        .header("x-api-key", api_key)
        .header("anthropic-version", "2023-06-01")
        .header("content-type", "application/json")
        .json(&serde_json::json!({
            "model": "claude-haiku-4-5",
            "max_tokens": 256,
            "system": system_prompt,
            "messages": [{ "role": "user", "content": user_prompt }],
        }))
        .send()
        .await?;

    let status = resp.status();
    let body_val: serde_json::Value = resp.json().await?;
    if !status.is_success() {
        anyhow::bail!("Anthropic API {status}: {body_val}");
    }
    let text = body_val["content"][0]["text"]
        .as_str()
        .ok_or_else(|| anyhow::anyhow!("unexpected Anthropic response shape: {body_val}"))?;

    Ok(text.trim().to_string())
}

/// Trigger LLM-backed derivation for one or more node UUIDs (non-blocking).
///
/// Returns immediately; per-uuid progress flows through the `derive:progress`
/// Tauri event. Plan 06-02's frontend listener consumes the stream and calls
/// `refreshNodes` after each `status: done`.
///
/// Guard ordering (matches 06-RESEARCH.md Pattern 4):
///   1. human_pinned  -> skipped-pinned (DERIVE-03; NO LLM call, NO write)
///   2. hash-skip     -> skipped         (DERIVE-02; NO LLM call, NO write)
///   3. LLM call      -> done | error
///
/// Errors are NEVER propagated to the caller's Result — the command always
/// returns Ok(()) and surfaces failures via `status: error` events so a single
/// failing uuid does not abort the whole batch.
#[tauri::command]
pub async fn derive_contracts(
    app: AppHandle,
    uuids: Vec<String>,
) -> Result<(), String> {
    let app_spawn = app.clone();
    tauri::async_runtime::spawn(async move {
        // 1. API key — read from env at invocation time (Pitfall 1 in 06-RESEARCH.md).
        let api_key = match std::env::var("ANTHROPIC_API_KEY") {
            Ok(k) if !k.trim().is_empty() => k,
            _ => {
                for uuid in &uuids {
                    emit_progress(
                        &app_spawn,
                        uuid,
                        "error",
                        Some(
                            "ANTHROPIC_API_KEY not set — set it in your shell (or ~/.zshenv \
                             for Finder-launched builds) and restart the app"
                                .into(),
                        ),
                    );
                }
                return;
            }
        };

        // 2. Repo path via the Plan 02-02 managed-state newtype. Same lookup
        //    shape as commands/mcp.rs launch_mcp_sidecar — Tauri state is keyed
        //    by the EXACT registered type, so we ask for `RepoState` and reach
        //    into `.0` for the inner Mutex.
        let repo_path_opt: Option<PathBuf> = app_spawn
            .try_state::<crate::commands::repo::RepoState>()
            .and_then(|state| state.0.lock().ok().and_then(|g| g.clone()));
        let Some(repo_path) = repo_path_opt else {
            for uuid in &uuids {
                emit_progress(
                    &app_spawn,
                    uuid,
                    "error",
                    Some("No repo open — pick a repository first".into()),
                );
            }
            return;
        };

        // 3. One reqwest Client shared across the batch (Anti-pattern:
        //    per-call clients). Client is Clone-cheap and pools connections.
        let http = match reqwest::Client::builder()
            .timeout(std::time::Duration::from_secs(60))
            .build()
        {
            Ok(c) => c,
            Err(e) => {
                for uuid in &uuids {
                    emit_progress(
                        &app_spawn,
                        uuid,
                        "error",
                        Some(format!("reqwest build: {e}")),
                    );
                }
                return;
            }
        };

        // 4. Sequential per-uuid loop. Pitfall 7 in 06-RESEARCH.md: sequential
        //    is fine for < 50 nodes at demo scale. A concurrency semaphore is
        //    Phase 9 polish — do NOT add it here.
        for uuid in &uuids {
            derive_one(&app_spawn, &http, &api_key, &repo_path, uuid).await;
        }
    });
    Ok(())
}

/// Single-node derivation. Emits `running` on entry and exactly one terminal
/// event (`done` | `skipped` | `skipped-pinned` | `error`) on exit.
async fn derive_one(
    app: &AppHandle,
    http: &reqwest::Client,
    api_key: &str,
    repo_path: &Path,
    uuid: &str,
) {
    emit_progress(app, uuid, "running", None);

    // ---- Load the sidecar ----
    let sidecar_path = repo_path.join(".contracts").join(format!("{uuid}.md"));
    let content = match std::fs::read_to_string(&sidecar_path) {
        Ok(c) => c,
        Err(e) => {
            emit_progress(
                app,
                uuid,
                "error",
                Some(format!("read {}: {e}", sidecar_path.display())),
            );
            return;
        }
    };
    let (mut fm, existing_body) = match parse_sidecar(&content) {
        Ok(pair) => pair,
        Err(e) => {
            emit_progress(app, uuid, "error", Some(format!("parse sidecar: {e}")));
            return;
        }
    };

    // ---- Guard 1: human-pinned (DERIVE-03) ----
    // Must check BEFORE the LLM call (Pitfall 6 ordering in 06-RESEARCH.md).
    if fm.human_pinned {
        emit_progress(app, uuid, "skipped-pinned", None);
        return;
    }

    // ---- Guard 2: hash-skip (DERIVE-02) ----
    // compute_code_hash returns None in two distinct cases:
    //   (a) code_ranges is EMPTY — legitimate for L0/L1 conceptual nodes that
    //       have no file backing. This is NOT an error; emit `skipped` so the
    //       UI shows "Skipped (no source)" and move on.
    //   (b) ANY referenced file is unreadable — this IS a real error and
    //       should surface to the user.
    if fm.code_ranges.is_empty() {
        emit_progress(
            app,
            uuid,
            "skipped",
            Some("no code_ranges — conceptual node, nothing to derive".into()),
        );
        return;
    }
    let Some(fresh_hash) = compute_code_hash(repo_path, &fm.code_ranges) else {
        emit_progress(
            app,
            uuid,
            "error",
            Some("cannot compute code_hash — referenced file unreadable".into()),
        );
        return;
    };
    let body_trimmed = existing_body.trim();
    if fm.code_hash.as_deref() == Some(fresh_hash.as_str()) && !body_trimmed.is_empty() {
        emit_progress(app, uuid, "skipped", None);
        return;
    }

    // ---- LLM call ----
    let source_snippet = extract_source(repo_path, &fm.code_ranges, SOURCE_SNIPPET_MAX_LINES);
    if source_snippet.trim().is_empty() {
        emit_progress(
            app,
            uuid,
            "error",
            Some("source snippet empty — cannot derive".into()),
        );
        return;
    }
    let file_label = fm
        .code_ranges
        .first()
        .map(|r| r.file.as_str())
        .unwrap_or("unknown");

    let llm_body = match call_anthropic_for_contract(
        http,
        api_key,
        &source_snippet,
        uuid,
        file_label,
        &fm.level,
        &fm.kind,
    )
    .await
    {
        Ok(b) => b,
        Err(e) => {
            emit_progress(app, uuid, "error", Some(format!("LLM call: {e}")));
            return;
        }
    };
    if llm_body.trim().is_empty() {
        emit_progress(
            app,
            uuid,
            "error",
            Some("LLM returned empty body".into()),
        );
        return;
    }

    // ---- Update frontmatter ----
    fm.code_hash = Some(fresh_hash);
    fm.contract_hash = Some(compute_contract_hash(&llm_body));
    fm.derived_at = Some(chrono::Utc::now().to_rfc3339());

    // ---- Atomic sidecar write (temp + rename) ----
    let new_content = match write_sidecar(&fm, &llm_body) {
        Ok(c) => c,
        Err(e) => {
            emit_progress(app, uuid, "error", Some(format!("write_sidecar: {e}")));
            return;
        }
    };
    let tmp = sidecar_path.with_extension("md.tmp");
    if let Err(e) = std::fs::write(&tmp, &new_content) {
        emit_progress(app, uuid, "error", Some(format!("write tmp: {e}")));
        return;
    }
    if let Err(e) = std::fs::rename(&tmp, &sidecar_path) {
        emit_progress(app, uuid, "error", Some(format!("rename: {e}")));
        return;
    }

    // ---- SQLite upsert (reuse Phase 2 helper — FTS5 rebuild included) ----
    let instances = app.state::<DbInstances>();
    let map = instances.0.read().await;
    let Some(db) = map.get("sqlite:contract-ide.db") else {
        emit_progress(app, uuid, "error", Some("DB not loaded".into()));
        return;
    };
    if let Err(e) = upsert_node_pub(db, &fm, &llm_body).await {
        emit_progress(app, uuid, "error", Some(format!("upsert: {e}")));
        return;
    }

    emit_progress(app, uuid, "done", None);
}
