**Step 1 — Rust IPC `find_substrate_by_intent`** in `commands/substrate.rs` (extending the file from plan 13-01).

Per 13-RESEARCH.md Open Question 1: a single Rust IPC aggregating contract FTS5 + substrate retrieval, returning unified results. Avoids MCP-sidecar hop for read-only queries.

```rust
#[derive(Debug, Serialize, Clone)]
#[serde(rename_all = "snake_case")]
pub struct IntentSearchHit {
    pub uuid: String,
    pub kind: String,                  // contract | constraint | decision | open_question | resolved_question | attempt | flow
    pub level: Option<String>,         // L0..L4 for contracts; null for substrate nodes
    pub name: String,
    pub summary: String,
    pub state: Option<String>,         // substrate state if applicable
    pub parent_uuid: Option<String>,   // for navigation: atom hit → parent surface
    pub score: f64,                    // ranking score (FTS5 rank or LLM rerank)
}

#[tauri::command]
pub async fn find_substrate_by_intent(
    db_instances: State<'_, DbInstances>,
    query: String,
    limit: Option<i32>,
) -> Result<Vec<IntentSearchHit>, String> {
    let limit = limit.unwrap_or(10).max(1).min(50);
    // 1. Query nodes_fts (Phase 5 contract FTS5) for contract+flow matches
    // 2. Query substrate_nodes (if exists) by name/summary FTS or LIKE
    // 3. Merge + dedupe by uuid; sort by score; take top `limit`
    // 4. Defensive: if substrate_nodes table absent, only return contract hits
    // ... (see implementation below)
}
```

**Implementation guidance:**
- Reuse the FTS5 pattern from Phase 5 Plan 05-02's `find_by_intent`. Look at existing MCP-sidecar `find_by_intent` SQL for nodes_fts.
- For substrate_nodes, use a basic LIKE query first (`WHERE name LIKE '%query%' OR summary LIKE '%query%'`); upgrade to FTS5 in a follow-up if precision is insufficient — Phase 11 may ship its own FTS for substrate_nodes.
- Score: contract FTS5 rank gives `bm25` or similar; substrate LIKE gives binary 1.0; sort contract hits first, then substrate hits, then truncate at `limit`. This is intentional: contracts are the primary navigation target; substrate nodes augment but should rank lower for the demo's flow→L2 / atom→L3 pattern.
- The return type is a fresh struct (NOT `SubstrateNodeSummary`) because hits include contracts (level field) and substrate nodes (state field) — different shapes warrant a unified `IntentSearchHit`.

Register in `lib.rs` `generate_handler!` with `commands::substrate::find_substrate_by_intent`.

**Step 3 — Build `IntentPalette.tsx`** as a sibling to the existing `CommandPalette.tsx`. Mirror its structure (cmdk `Command.Dialog`), but:
- Cmd+P binding instead of Cmd+K (use `e.key === 'p' && (e.metaKey || e.ctrlKey)`)
- `e.preventDefault()` BEFORE `setOpen` per Pitfall 2 in research (Cmd+P would otherwise trigger macOS Print dialog)
- Input drives a debounced 300ms async query (not a static action filter)
- Empty-state message when no query
- Loading state during fetch (disable input or show spinner)

```tsx
import { Command } from 'cmdk';
import { useEffect, useState } from 'react';
import { findSubstrateByIntent, type IntentSearchHit } from '@/ipc/substrate';
import { useGraphStore } from '@/store/graph';
import { useSidebarStore } from '@/store/sidebar';
import { IntentPaletteHit } from './IntentPaletteHit';

export function IntentPalette() {
  const [open, setOpen] = useState(false);
  const [query, setQuery] = useState('');
  const [hits, setHits] = useState<IntentSearchHit[]>([]);
  const [loading, setLoading] = useState(false);

  // Cmd+P binding — preventDefault BEFORE setOpen (Pitfall 2)
  useEffect(() => {
    const onKey = (e: KeyboardEvent) => {
      if (e.key === 'p' && (e.metaKey || e.ctrlKey)) {
        e.preventDefault();
        setOpen((o) => !o);
      }
    };
    document.addEventListener('keydown', onKey);
    return () => document.removeEventListener('keydown', onKey);
  }, []);

  // Debounced query (300ms)
  useEffect(() => {
    if (!open || !query.trim()) {
      setHits([]);
      return;
    }
    setLoading(true);
    const handle = setTimeout(async () => {
      try {
        const result = await findSubstrateByIntent(query, 10);
        setHits(result);
      } catch (err) {
        console.error('[IntentPalette] query failed:', err);
        setHits([]);
      } finally {
        setLoading(false);
      }
    }, 300);
    return () => clearTimeout(handle);
  }, [query, open]);

  function handleSelect(hit: IntentSearchHit) {
    setOpen(false);
    setQuery('');
    if (hit.kind === 'flow') {
      // Flow → land at L2 vertical-chain view
      useSidebarStore.getState().setSelectedFlow(hit.uuid);
      useGraphStore.getState().pushParent(hit.uuid);
    } else if (hit.kind === 'contract' && hit.level === 'L4') {
      // Atom contract → land at L3 trigger view of parent + auto-focus the chip
      const parent = hit.parent_uuid;
      if (parent) {
        useGraphStore.getState().pushParent(parent);
      }
      // Set focused atom uuid in graph store — plan 13-05 ScreenCard reads this to halo the chip
      useGraphStore.setState({ focusedAtomUuid: hit.uuid });
    } else if (hit.kind === 'contract') {
      // Other contract levels → just navigate
      useGraphStore.getState().pushParent(hit.uuid);
    } else {
      // Substrate node → open inspector at parent (or in plan 13-07's archaeology modal)
      // For now: select the parent contract if present
      if (hit.parent_uuid) {
        useGraphStore.setState({ selectedNodeUuid: hit.parent_uuid });
      }
    }
  }

  return (
    <Command.Dialog open={open} onOpenChange={setOpen} label="Find by intent">
      <Command.Input
        value={query}
        onValueChange={setQuery}
        placeholder="Search by intent..."
        autoFocus
      />
      <Command.List>
        {loading && <Command.Loading>Searching...</Command.Loading>}
        {!loading && query && hits.length === 0 && (
          <Command.Empty>No matches.</Command.Empty>
        )}
        {hits.map((hit) => (
          <Command.Item key={hit.uuid} value={hit.uuid} onSelect={() => handleSelect(hit)}>
            <IntentPaletteHit hit={hit} />
          </Command.Item>
        ))}
      </Command.List>
    </Command.Dialog>
  );
}
```

**Notes on `useGraphStore` actions:** The exact action names depend on Phase 3's `graphStore` shape. Look at `store/graph.ts` for `pushParent`, `popParent`, `setParentUuidStack`. If `focusedAtomUuid` doesn't exist yet, ADD it as a slice (`focusedAtomUuid: string | null` + `setFocusedAtomUuid` action). Plan 13-05 (ScreenCard) reads it to halo the chip.
