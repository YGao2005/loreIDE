**Step 1 — Create `contract-ide/src/store/substrate.ts` (mirror useDriftStore pattern verbatim).**

Type: `SubstrateNodeState = 'fresh' | 'stale' | 'superseded' | 'intent_drifted'`. Per 13-RESEARCH.md Pattern 1, the store keys by uuid (string) and stores the state enum:

```typescript
import { create } from 'zustand';

export type SubstrateNodeState = 'fresh' | 'stale' | 'superseded' | 'intent_drifted';

interface SubstrateState {
  nodeStates: Map<string, SubstrateNodeState>;
  setNodeState: (uuid: string, state: SubstrateNodeState) => void;
  bulkSet: (updates: { uuid: string; state: SubstrateNodeState }[]) => void;
  clearNodeState: (uuid: string) => void;
  reset: () => void;
}

export const useSubstrateStore = create<SubstrateState>((set) => ({
  nodeStates: new Map(),
  setNodeState: (uuid, state) =>
    set((s) => {
      const next = new Map(s.nodeStates);
      next.set(uuid, state);
      return { nodeStates: next };
    }),
  bulkSet: (updates) =>
    set((s) => {
      const next = new Map(s.nodeStates);
      for (const { uuid, state } of updates) next.set(uuid, state);
      return { nodeStates: next };
    }),
  clearNodeState: (uuid) =>
    set((s) => {
      if (!s.nodeStates.has(uuid)) return s;
      const next = new Map(s.nodeStates);
      next.delete(uuid);
      return { nodeStates: next };
    }),
  reset: () => set({ nodeStates: new Map() }),
}));
```

**Why this exact shape:** `useDriftStore` uses Set<string>; rollup stores use Map<string, RollupState>. The substrate store needs Map (multi-valued) like rollup, not Set (boolean) like drift. The `Map` identity-change is what triggers Zustand selector subscribers — DO NOT mutate in place.

#[tauri::command]
pub async fn get_substrate_states_for_canvas(
    db_instances: State<'_, DbInstances>,
) -> Result<Vec<SubstrateNodeSummary>, String> {
    let instances = db_instances.0.read().await;
    let pool_ref = instances
        .get("sqlite:contract-ide.db")
        .ok_or_else(|| "DB not loaded".to_string())?;

    let pool = match pool_ref {
        tauri_plugin_sql::DbPool::Sqlite(p) => p,
        #[allow(unreachable_patterns)]
        _ => return Err("Unsupported pool".into()),
    };

    // Defensive: substrate_nodes table is created by Phase 11 distiller migration.
    // If not present yet, return empty Vec so Phase 13 doesn't gate on Phase 11.
    let exists: (i64,) = sqlx::query_as(
        "SELECT COUNT(*) FROM sqlite_master WHERE type='table' AND name='substrate_nodes'"
    )
    .fetch_one(pool)
    .await
    .map_err(|e| e.to_string())?;
    if exists.0 == 0 {
        return Ok(vec![]);
    }

    let rows = sqlx::query(
        r#"SELECT uuid, kind, COALESCE(state, 'fresh') as state, name,
                  COALESCE(summary, '') as summary, session_id, turn_ref,
                  verbatim_quote, actor, confidence
           FROM substrate_nodes"#,
    )
    .fetch_all(pool)
    .await
    .map_err(|e| e.to_string())?;

    Ok(rows
        .into_iter()
        .map(|r| SubstrateNodeSummary {
            uuid: r.get("uuid"),
            kind: r.get("kind"),
            state: r.get("state"),
            name: r.get("name"),
            summary: r.get("summary"),
            session_id: r.try_get("session_id").ok(),
            turn_ref: r.try_get("turn_ref").ok(),
            verbatim_quote: r.try_get("verbatim_quote").ok(),
            actor: r.try_get("actor").ok(),
            confidence: r.try_get("confidence").ok(),
        })
        .collect())
}

**Step 2 — Add `resolveNodeState` helper.** Add at the top of `contractNodeStyles.ts` (or in a new sibling file `resolveNodeState.ts` if cleaner):

```typescript
import type { SubstrateNodeState } from '@/store/substrate';

export type NodeVisualState =
  | 'healthy'
  | 'drifted'
  | 'rollup_stale'
  | 'rollup_untracked'
  | 'intent_drifted'
  | 'superseded';

export function resolveNodeState(
  uuid: string,
  driftedUuids: Set<string>,
  rollupStaleUuids: Set<string>,
  untrackedUuids: Set<string>,
  substrateStates: Map<string, SubstrateNodeState>,
): NodeVisualState {
  // Precedence per ROADMAP SC 6 + Phase 8 PROP-02 token compatibility:
  //   red (drifted)  >  orange (intent_drifted)  >  amber (rollup_stale)
  //                  >  orange-muted (superseded) >  gray (rollup_untracked)  >  healthy
  if (driftedUuids.has(uuid)) return 'drifted';
  const sub = substrateStates.get(uuid);
  if (sub === 'intent_drifted') return 'intent_drifted';
  if (rollupStaleUuids.has(uuid)) return 'rollup_stale';
  if (sub === 'superseded') return 'superseded';
  if (untrackedUuids.has(uuid)) return 'rollup_untracked';
  return 'healthy';
}
```

**Why this precedence and not another:** Phase 7 ships drift = red (highest); Phase 8 ships rollup_stale = amber + rollup_untracked = gray. Phase 13's intent_drifted = orange must sit BETWEEN red and amber per ROADMAP SC 6 ("precedence red > orange > amber > gray"). The `superseded` state is a softer signal than amber (the priority shift didn't hit yet, but a related decision did) — it lives between amber and gray.
