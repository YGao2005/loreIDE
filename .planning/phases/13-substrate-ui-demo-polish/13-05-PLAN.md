**Step 1 — `src/lib/iframeChipPositioning.ts` — postMessage protocol.**

The iframe (loaded from localhost — same origin per Plan 04-03 frame-src CSP) responds to a `request-chip-rects` message with positions of all `[data-contract-uuid]` elements.

**Critical caveat:** The iframe's content runs the user's Next.js dev server. We do NOT control its scripts. We CAN inject a parent-layer listener that polls the iframe's DOM via `postMessage` AND defensively fall back to direct `iframe.contentDocument.querySelectorAll(...)` when same-origin (which it IS for localhost dev). The postMessage path is the canonical contract; the direct DOM access is the fallback used until BABEL-01 ships an iframe-side responder.

```typescript
export interface ChipRect {
  uuid: string;
  rect: { top: number; left: number; width: number; height: number };
}

export interface ChipRectMessage {
  type: 'chip-rects';
  rects: ChipRect[];
}

/**
 * Query the iframe DOM for all elements with data-contract-uuid.
 *
 * Strategy:
 * 1. Try same-origin direct access (iframe.contentDocument). Works for localhost dev (Phase 4 CSP).
 * 2. Fallback: postMessage 'request-chip-rects'; iframe-side responder posts back ChipRectMessage.
 * 3. If both fail, return empty array. Caller renders chip-less iframe (still useful for layout).
 */
export async function requestChipRects(iframe: HTMLIFrameElement, timeoutMs = 250): Promise<ChipRect[]> {
  // Same-origin direct path (preferred, no protocol coordination needed)
  try {
    const doc = iframe.contentDocument;
    if (doc) {
      const elems = doc.querySelectorAll<HTMLElement>('[data-contract-uuid]');
      const rects: ChipRect[] = [];
      const iframeRect = iframe.getBoundingClientRect();
      elems.forEach((el) => {
        const r = el.getBoundingClientRect();
        const uuid = el.getAttribute('data-contract-uuid');
        if (!uuid) return;
        rects.push({
          uuid,
          rect: {
            top: r.top - iframeRect.top,    // chip coordinates relative to iframe top-left
            left: r.left - iframeRect.left,
            width: r.width,
            height: r.height,
          },
        });
      });
      return rects;
    }
  } catch {
    // SecurityError (cross-origin): fall through to postMessage path
  }

  // postMessage path — iframe must register a 'request-chip-rects' listener
  return new Promise<ChipRect[]>((resolve) => {
    const handle = setTimeout(() => {
      window.removeEventListener('message', listener);
      resolve([]);
    }, timeoutMs);
    const listener = (e: MessageEvent) => {
      if (e.source !== iframe.contentWindow) return;
      const data = e.data as ChipRectMessage;
      if (data?.type === 'chip-rects') {
        clearTimeout(handle);
        window.removeEventListener('message', listener);
        resolve(data.rects);
      }
    };
    window.addEventListener('message', listener);
    iframe.contentWindow?.postMessage({ type: 'request-chip-rects' }, '*');
  });
}
```

**Why both paths:** Same-origin works today (Plan 04-03 CSP allows http://localhost:* in frame-src). If a future scenario serves the iframe from a different origin, postMessage path gracefully degrades. Document inline.

**Step 4 — `AtomChipOverlay.tsx` — overlay container.**

Mounts as an absolute-positioned div over the iframe. On iframe load + on window resize, queries chip rects via the helper. Maintains a state list of {uuid, name, rect}.

```tsx
import { useEffect, useState, useRef } from 'react';
import { requestChipRects } from '@/lib/iframeChipPositioning';
import { AtomChip } from './AtomChip';
import { useGraphStore } from '@/store/graph';

interface Props {
  iframeRef: React.RefObject<HTMLIFrameElement>;
  parentUuid: string; // the screen contract uuid; used to filter atoms by parent_uuid
}

export function AtomChipOverlay({ iframeRef, parentUuid }: Props) {
  const [chips, setChips] = useState<Array<{ uuid: string; name: string; rect: { top: number; left: number; width: number; height: number } }>>([]);

  // Subscribe to graph store atoms whose parent_uuid matches
  const atoms = useGraphStore((s) =>
    s.nodes.filter((n) => n.parent_uuid === parentUuid && n.level === 'L4')
  );

  const refresh = async () => {
    const iframe = iframeRef.current;
    if (!iframe) return;
    const rects = await requestChipRects(iframe);
    // Join rects with atom names by uuid
    const merged = rects.map((r) => {
      const atom = atoms.find((a) => a.uuid === r.uuid);
      return { uuid: r.uuid, name: atom?.name ?? r.uuid.slice(0, 8), rect: r.rect };
    });
    setChips(merged);
  };

  useEffect(() => {
    const iframe = iframeRef.current;
    if (!iframe) return;
    const onLoad = () => refresh();
    const onResize = () => refresh();
    iframe.addEventListener('load', onLoad);
    window.addEventListener('resize', onResize);
    // Initial fetch with small delay to let any HMR settle
    const initialHandle = setTimeout(refresh, 200);
    return () => {
      clearTimeout(initialHandle);
      iframe.removeEventListener('load', onLoad);
      window.removeEventListener('resize', onResize);
    };
  }, [atoms.length]); // re-bind if atoms change

  return (
    <div className="absolute inset-0 pointer-events-none" data-overlay="atom-chips">
      {chips.map((c) => (
        <AtomChip key={c.uuid} uuid={c.uuid} name={c.name} rect={c.rect} />
      ))}
    </div>
  );
}
```

**Why pointer-events-none on container, pointer-events-auto on chip:** Chips intercept clicks (Inspect mode default). The container's empty space passes clicks through to iframe (Interact mode behavior). When ROADMAP optional polish "Inspect/Interact toggle" lands, only the chip's pointer-events flips.

**Step 1 — `ScreenCard.tsx` — UI mode L3 trigger card.**

Reuses Phase 4 PreviewTab's iframe-loading pattern (probe + iframe + retry). The new wrinkle is the AtomChipOverlay layered on top.

```tsx
import { memo, useEffect, useRef, useState } from 'react';
import type { NodeProps } from '@xyflow/react';
import { Handle, Position } from '@xyflow/react';
import { invoke } from '@tauri-apps/api/core';
import { screenCardStyles } from './screenCardStyles';
import { AtomChipOverlay } from './AtomChipOverlay';
import { resolveNodeState } from './contractNodeStyles';
import { useDriftStore } from '@/store/drift';
import { useRollupStore } from '@/store/rollup';
import { useSubstrateStore } from '@/store/substrate';
import { useGraphStore } from '@/store/graph';

export interface ScreenCardData extends Record<string, unknown> {
  uuid: string;
  name: string;        // 'Account Settings'
  route: string;       // '/account/settings'
  devServerUrl?: string; // 'http://localhost:3000' default
}

type LoadState = 'probing' | 'unreachable' | 'reachable';

function ScreenCardImpl({ data }: NodeProps) {
  const d = data as ScreenCardData;
  const baseUrl = d.devServerUrl ?? 'http://localhost:3000';
  const fullUrl = `${baseUrl}${d.route}`;
  const iframeRef = useRef<HTMLIFrameElement>(null);
  const [loadState, setLoadState] = useState<LoadState>('probing');
  const [interactMode, setInteractMode] = useState(false); // ROADMAP optional polish: Inspect (default) vs Interact

  const drifted = useDriftStore((s) => s.driftedUuids);
  const rollupStale = useRollupStore((s) => s.rollupStaleUuids);
  const untracked = useRollupStore((s) => s.untrackedUuids);
  const substrate = useSubstrateStore((s) => s.nodeStates);
  const state = resolveNodeState(d.uuid, drifted, rollupStale, untracked, substrate);

  useEffect(() => {
    let cancelled = false;
    invoke<boolean>('probe_route', { url: fullUrl })
      .then((reachable) => {
        if (cancelled) return;
        setLoadState(reachable ? 'reachable' : 'unreachable');
      })
      .catch(() => !cancelled && setLoadState('unreachable'));
    return () => {
      cancelled = true;
    };
  }, [fullUrl]);

  return (
    <div className={screenCardStyles({ state })} style={{ width: 600, minHeight: 400 }}>
      <Handle type="target" position={Position.Top} />
      <header className="flex items-center justify-between px-3 py-2 border-b border-border/40 bg-muted/30">
        <div className="flex items-center gap-2">
          <code className="text-xs font-mono text-muted-foreground">{d.route}</code>
          <span className="text-sm font-medium">{d.name}</span>
        </div>
        <button
          onClick={() => setInteractMode((m) => !m)}
          className="text-[10px] px-2 py-0.5 rounded border border-border/50 hover:bg-muted/50"
          title="Inspect mode: chips intercept clicks. Interact mode: iframe receives clicks."
        >
          {interactMode ? 'Interact' : 'Inspect'}
        </button>
      </header>
      <div className="relative" style={{ height: 'calc(100% - 36px)', minHeight: 360 }}>
        {loadState === 'reachable' && (
          <>
            <iframe
              ref={iframeRef}
              src={fullUrl}
              sandbox="allow-scripts allow-same-origin allow-forms"
              className="w-full h-full"
              style={{ border: 0, pointerEvents: interactMode ? 'auto' : 'none' }}
              title={d.name}
            />
            <AtomChipOverlay iframeRef={iframeRef} parentUuid={d.uuid} />
          </>
        )}
        {loadState === 'probing' && (
          <div className="flex items-center justify-center h-full text-muted-foreground text-sm">
            Probing dev server at {baseUrl}...
          </div>
        )}
        {loadState === 'unreachable' && (
          <div className="flex flex-col items-center justify-center h-full gap-2 text-sm">
            <div className="text-muted-foreground">Dev server unreachable at {baseUrl}</div>
            <button
              onClick={() => setLoadState('probing')}
              className="text-xs px-3 py-1 rounded bg-primary text-primary-foreground"
            >
              Retry
            </button>
          </div>
        )}
      </div>
      <Handle type="source" position={Position.Bottom} />
    </div>
  );
}

export const ScreenCard = memo(ScreenCardImpl);
```

**Why iframe pointer-events toggle:** In Inspect mode (default), the iframe's `pointer-events: none` AND chips have `pointer-events: auto` — clicks on a chip work; clicks on iframe whitespace are no-ops. In Interact mode, iframe takes pointer events and chips fade. Toggle button in header. Per ROADMAP "optional polish — listed as polish task; demo doesn't require it."
