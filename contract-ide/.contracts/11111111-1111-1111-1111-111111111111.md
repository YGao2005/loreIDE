## Intent
When an automated session has just written a fresh, source-derived description of a node in the contract graph, this is the safe way to publish that description back into the project. It refuses to overwrite any node a human has explicitly claimed as their own, so machine-generated text never silently replaces human authorship.

## Role
Sits between the deriving session and the on-disk contract for one node, performing the write only when policy allows it.

## Inputs
- uuid: string — identifier of the node whose contract is being rewritten
- body: string — the freshly generated contract body, without the leading header block

## Outputs
- On success: a confirmation string of the form `DONE: <uuid> — code_hash=<hex|∅>, contract_hash=<12-char prefix>…, derived_at=<ISO-8601>. Rust watcher will refresh SQLite within 2s.`
- On pinned-guard hit: `SKIPPED-PINNED: <uuid> is human_pinned — sidecar left unchanged.`
- On error: `ERROR: <reason>` describing missing repo path, missing sidecar, malformed frontmatter, unreadable source files, or empty body

## Invariants
- A node whose frontmatter carries `human_pinned: true` is never overwritten (lines 64–68).
- The frontmatter `uuid` field is preserved verbatim across the write (line 87).
- `code_hash` is recomputed over the current contents of every line range in `code_ranges` before the write lands (lines 74, 100–119).
- `contract_hash` is recomputed over the trimmed body that is actually persisted (line 84).
- `derived_at` is set to the wall-clock time of THIS write, not carried over from any prior write (line 85).
- An empty body (after trimming) is rejected before any file is touched (line 71).
- The on-disk file is replaced via write-then-rename, so a partial write cannot leave a half-written file visible to readers (lines 91–93).
- A `code_hash` of `null` is permitted only when `code_ranges` is empty; otherwise unreadable source aborts the write (lines 75–79, 100–101).

## Side Effects
- Reads the existing sidecar file at `<repo>/.contracts/<uuid>.md`.
- Reads each source file referenced in `code_ranges` to recompute the code hash.
- Writes a temp file `<sidecar>.tmp-<pid>` then renames it over the sidecar — atomic on POSIX filesystems.
- Touches the sidecar's mtime, which the Rust fs watcher picks up and propagates into SQLite within ~2s.

## Failure Modes
- Repo path not configured → `ERROR: <message from getRepoPath>`.
- Sidecar file missing → `ERROR: sidecar not found: <path>`.
- Frontmatter fence cannot be split → `ERROR: failed to split frontmatter fence at <path>`.
- YAML parse failure → `ERROR: YAML parse failed at <path>: <reason>`.
- Frontmatter missing or malformed `uuid` → `ERROR: parsed frontmatter missing uuid at <path>`.
- One or more files in `code_ranges` cannot be read → `ERROR: cannot read source for <uuid> — one or more files in code_ranges missing`.
- Body is empty after trimming → `ERROR: empty body supplied for <uuid>`.

## HTTP
Transport: MCP stdio
Tool name: write_derived_contract
Auth: none — runs in-process with the local IDE

## Examples
GIVEN a node's contract has not been pinned by a human and every source file it points to exists on disk
WHEN  an automated derivation session submits a fresh contract body for that node
THEN  the contract on disk is replaced with the new body
  AND the caller is told the contract was updated, with a short receipt of which source state and which body it now reflects
  AND the IDE graph reflects the new contract within a couple of seconds

GIVEN a node's contract has been pinned by a human as the authoritative version
WHEN  an automated derivation session submits a new body for that same node
THEN  the contract on disk is left exactly as the human left it
  AND the writer is told the contract is pinned and was left unchanged

GIVEN a node points to a source file that has been moved or deleted since the contract was last written
WHEN  an automated derivation session submits a new body for that node
THEN  the contract on disk is left untouched
  AND the writer is told the update failed, naming the missing source as the cause
