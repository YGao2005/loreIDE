## Intent
Gives the person viewing a node in the inspector an at-a-glance signal of whether its written contract still matches the code it describes, or has fallen out of sync. When the contract is out of date, it offers a one-click path to reconcile. When the contract has never been derived, it says so plainly rather than pretending everything is fine.

## Role
A status pill rendered inside the inspector's header strip, visible across every inspector tab for the selected node.

## Inputs
- `node: ContractNode | null` — the node the inspector is currently focused on, or nothing if the inspector is empty.
- `onReconcile?: () => void` — optional handler the parent can register to start bringing a stale contract back in line with its source code.

## Outputs
- A small status pill rendered into the inspector's header, or nothing when no node is selected. The pill takes one of three visible forms: a neutral "Not derived" label for nodes that have never been derived, a green "Synced" badge when the contract matches the code, or a red "Drifted" badge with a Reconcile action when the two have diverged.

## Invariants
- Nothing is rendered when no node is selected (line 39).
- A node missing either side of the match signal is labeled "Not derived" in a neutral, non-alarming style (lines 21, 42–48).
- The green "Synced" badge is shown only when both sides of the match signal are present and agree (lines 21–22, 66).
- The red "Drifted" badge is shown only when both sides of the match signal are present and disagree (lines 21–22, 50, 66).
- The Reconcile action is offered only in the drifted state, never alongside the synced or untracked badges (lines 67–81).
- When the parent has not wired up a reconcile handler, clicking Reconcile surfaces a transient placeholder hint rather than silently doing nothing (lines 71–77).
- The placeholder hint clears itself automatically after a short moment rather than lingering (lines 33–37).

## Examples
GIVEN a node whose written contract still matches its current source code
WHEN the inspector opens that node
THEN a green "Synced" pill appears in the inspector's header

GIVEN a node whose source code has been edited since its contract was last written
WHEN the inspector opens that node
THEN a red "Drifted" pill appears in the inspector's header
  AND a Reconcile link appears next to it inviting the user to bring the contract back in line

GIVEN the inspector is showing a drifted node and no reconcile flow has been wired up yet
WHEN the user clicks Reconcile
THEN a short note appears saying the full reconcile flow is still pending
  AND the note clears itself a moment later without any further action from the user

## Interaction
The Reconcile link is the only interactive element, and only in the drifted state. Clicking it invokes the parent's reconcile handler when one is registered; otherwise it surfaces a brief self-dismissing hint that the full reconcile experience is not yet available.

## Visual States
- Hidden — no node is selected; the component renders nothing.
- Untracked — small neutral "Not derived" label for nodes that have never been derived.
- Synced — green pill with a solid green dot and the word "Synced".
- Drifted — red pill with a pulsing red dot, the word "Drifted", and an inline Reconcile link.
- Drifted with hint — the drifted pill plus a trailing amber note that the reconcile flow ships in a later phase; shown briefly after clicking Reconcile without a registered handler.
